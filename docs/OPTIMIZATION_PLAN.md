# 《暗影深渊》优化路线图

> 目标：从 **可玩原型（Pre-Alpha）** 提升到 **可对外测试的 Beta / 可发行候选版**  
> 周期参考：单人全职约 **4～6 个月**；业余开发约 **8～12 个月**

---

## 一、总体策略

商业放置 RPG 的竞争力来自四件事：

1. **前 30 分钟** — 玩家是否觉得「好玩、看得懂、有成长」
2. **前 7 天** — 玩家是否每天想打开
3. **前 30 小时** — 内容是否撑得住、数值是否不腻
4. **发行层** — 稳定性、存档、基础商业化

建议路线：**先体验与战斗爽感 → 再内容与数值 → 再留存与运营 → 最后发行工程**。

```
P0 体验修复 → P1 战斗爽感 → P2 内容数值 → P3 留存引导 → P4 美术音频 → P5 工程发行
```

## 二、阶段划分

### 阶段 0：基础体验修复（1～2 周）— 必做

| # | 任务 | 交付物 | 验收标准 |
|---|------|--------|----------|
| 0.1 | 新手引导（FTUE） | 5～8 步遮罩引导 | 新存档 3 分钟内完成首次装备 + Boss 指引 |
| 0.2 | 战斗 UI 定稿 | 固定布局规范 | 720×1280 与常见长屏无遮挡 |
| 0.3 | 技能栏战斗 HUD | 技能图标 + CD 环形/遮罩进度 | 放技能时玩家能看到冷却 |
| 0.4 | 资源清理收尾 | 删除未引用资源；`AssetRegistry` 统一入口 | 无死链贴图 |
| 0.5 | README + 版本号 | 玩法简介、构建方式、changelog | 新人 10 分钟能跑起来 |

**完成标志**：3 个新玩家试玩，不问操作也能完成第一轮循环。

**进度**：0.3 技能 CD 条、1.6 伤害飘字 ✅ 已实现。

### 阶段 2：内容与数值长线（3～4 周）

#### 内容扩展

| 内容 | 现状 | 建议目标 |
|------|------|----------|
| 区域 | 10 个 | 每区 3 种波次模板（普通/精英/混合） |
| 敌人 | 名称映射 | 每区 2 专属贴图 + 1 精英变体 |
| Boss | 10 个 | 每 Boss 专属入场/技能特效 |
| 深渊塔 | 有框架 | 50～100 层，每 10 层保底奖励 |
| 装备 | 随机生成 | 每 10 级 1 套套装 + 2 件套效果 |

#### 数值外置

```
data/
  zones.json
  enemies.json
  skills.json
  equipment_tables.json
  drop_tables.json
  balance_curves.json
```

`DataManager` 启动时加载 JSON，调平衡只改表不改代码。

#### 数值验收指标

| 指标 | 目标 |
|------|------|
| 同等级一波战斗时长 | 15～40 秒 |
| 死亡频率（同区） | 每 10 波 ≤ 1 次 |
| Boss 首杀尝试次数 | 2～4 次 |
| 升级间隔（中期） | 3～8 分钟一级 |

### 阶段 5：留存与每日循环（2 周）

| 系统 | 建议 |
|------|------|
| 每日任务 | 3 简单 + 1 挑战，全完成给宝箱 |
| 7 日签到 | 第 7 天宠物蛋或紫装 |
| 离线收益 | 弹窗展示并一键领取 |
| 成就 | 每 5 个成就给天赋点或宝石 |
| 商店 | 每日刷新 3 件折扣货 |

**核心公式**：领离线 → 领每日 → 消红点 → 打 1 波/Boss → 感到变强 → 退出（3～8 分钟）。

## 三、优先级矩阵

```
                    影响大
                      ↑
    P0 引导/战斗HUD   |   P1 战斗爽感/飘字/BGM
    P0 UI重叠修复     |   P2 数值外置
                      |
    ──────────────────┼──────────────────→ 成本低
                      |
    P5 云存档/IAP     |   P4 全资产重绘
                      ↓
                    影响小
```

**时间紧时优先 10 项**：

1. 新手引导（8 步）
2. 战斗技能 CD 图标 ✅
3. 伤害飘字 ✅
4. 真实音效 + 1 首 BGM

7. 数值 JSON 外置

## 六、风险与原则

1. **不要先堆系统** — 先打磨已有循环。
2. **不要先商业化** — 体验不稳时接 IAP 只会差评。
3. **美术可渐进** — 先统一规范，再逐张替换。
4. **数值一定要外置** — 后续平衡的根基。
5. **每周录屏试玩 15 分钟** — 比看代码更能发现问题。
