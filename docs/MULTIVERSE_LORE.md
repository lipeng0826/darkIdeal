# 万界裂隙 — 多元宇宙世界观

> 战斗不是「框框干」，而是**穿越一个个破碎世界**；每个区域是一界，Boss 是界锚守护者。

---

## 核心设定

| 概念 | 说明 |
|------|------|
| **万界裂隙** | 吞噬诸多元宇宙的灾难，把不同世界撕成碎片 |
| **裂隙行者** | 玩家身份，能在各界短暂锚定、战斗、带走界尘（材料） |
| **界域** | 对应游戏内 10 个 Zone，每个是一枚世界残片 |
| **界锚守护者** | Boss，击破后获得该界 **界锚残片**（见 [界锚残片玩法](HORCRUX_LOOP.md)） |
| **纪元天书** | 集齐残片后的终局目标，封住万界裂隙 |

开场白见 `data/multiverse_lore.json` → `premise`。

## 玩家体验节奏

```
进入游戏 → 开场「坠入裂隙」
    ↓
首次进入某界 → 穿越语 Toast + 战斗日志世界观介绍
    ↓
击败 Boss → 获得界锚残片 + Boss 传说 + 解锁下一界域
    ↓
世界地图 → 显示「万界裂隙」标题 + 各节点多元宇宙名
    ↓
战斗面板 → 界域名 + 类型徽章 + 纪元 + 一句话 tagline
```

## 代码入口

| 模块 | 职责 |
|------|------|
| `data/multiverse_lore.json` | 全部世界观数据 |
| `LoreManager` | 加载、查询、触发叙事事件 |
| `save.lore` | `visited_realms`、`seen_first_game` |
| `GameManager` | Boss 解锁 / 切区时调用 LoreManager |
| `main_ui` / `adventure_map_view` | 显示界域名与类型 |

### 扩展新界域

1. 在 `multiverse_lore.json` 的 `realms` 追加一条（`zone_index` 对应 DataManager.ZONES）
2. 如需新类型，在 `dimension_types` 追加
3. 后续可为每界配专属战斗背景、敌人皮肤、BGM（美术阶段）

### 扩展叙事

- 改 `travel_text` / `intro` → 首次进入该界时播放
- 改 `boss_lore` → Boss 击杀后战斗日志
- 加 `rift_events` → 全局事件触发文案

## 与养成系统的衔接

- **材料名称**可逐步改成「界尘」风格（当前保留暗影精华等，lore 里有释义）
- **Lv.90 深渊天书** = 集齐界锚残片后，在深渊枢纽重铸纪元天书（见 [HORCRUX_LOOP.md](HORCRUX_LOOP.md)）
- **转生 Lv.50** = 裂隙行者第一次「轮回」，带着诸界记忆重来

| 界 | 背景方向 | 敌人方向 |
|----|----------|----------|
| 汉末遗冢 | 烽烟古墓、断戟残旗 | 黄巾尸兵、军阀亡魂 |
